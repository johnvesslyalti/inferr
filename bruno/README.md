# Bruno API Testing Collection

This is a Bruno collection for testing the RAG API endpoints.

## Setup

### 1. Install Bruno

Download and install Bruno from [bruno.so](https://www.bruno.so)

### 2. Open Collection

1. Open Bruno
2. Click "Open Collection"
3. Navigate to this `bruno` folder
4. Select the collection

### 3. Select Environment

Click the environment dropdown and select:
- **local** - For local development (http://localhost:3001)
- **production** - For production deployment

## Requests

### 1. Health Check
Check if the RAG API is running

```
GET /rag/health
```

### 2. Initialize Demo Data
Load 5 demo documents into the vector store

```
POST /rag/init
```

Response includes all loaded documents with their content.

### 3. Get Documents
Retrieve all documents currently loaded in the vector store

```
GET /rag/documents
```

### 4. Query - What is Machine Learning?
Example query to test the RAG pipeline

```
POST /rag/query
Body: { "query": "What is machine learning?" }
```

Response includes:
- Answer generated by Claude
- Retrieved documents used for context

### 5. Query - Explain Neural Networks
Another example query

```
POST /rag/query
Body: { "query": "Explain neural networks and how they work" }
```

### 6. Query - What are Transformers?
Test retrieval of transformer documentation

```
POST /rag/query
Body: { "query": "What are transformers and what are they used for in AI?" }
```

## Custom Queries

To test custom queries:

1. Duplicate any query request
2. Change the `query` value in the body
3. Send the request

Example queries to try:
- "What is deep learning?"
- "Tell me about natural language processing"
- "How do neural networks learn?"
- "What are the applications of transformers?"

## Environment Variables

### Local Environment

```
baseUrl: http://localhost:3001
```

### Production Environment

```
baseUrl: https://api.yourdomain.com
```

Update the `production.bru` file with your actual API domain.

## Notes

- Ensure the API is running before making requests
- The `/rag/init` endpoint initializes demo data (safe to call multiple times)
- Query endpoints require `ANTHROPIC_API_KEY` to be set in the API's `.env` file
- Retrieved documents are returned with each query to show which data was used for context

## Tips

- Use Bruno's environment switcher to quickly toggle between local and production
- All requests are pre-configured with necessary headers
- Response times vary based on API key and Claude API availability
